# Meeting Survival

## The Core Principle

A meeting is not a place where decisions are made. It is a place where the need for another meeting is established.

Your goal: leave every meeting without any action items assigned to you.

---

## Pre-Meeting Preparation

### The Conflict Check

Before accepting any meeting invite, run this check:

```
Does this meeting have:
├─ A clear agenda?      → Still attend (you need the social credit)
├─ No clear agenda?     → Attend (easier to avoid action items in chaos)
├─ Your name in the title? → Mandatory. But you can still dodge work.
└─ "Quick sync" in the title? → It is never quick. Block 1 hour after it.
```

### The Strategic Late Arrival

Arriving 3-4 minutes late has two benefits:
1. The awkward small talk phase is over
2. You missed any pre-meeting action item volunteering

Do not be more than 5 minutes late. That crosses from "strategic" into "noticed."

## In-Meeting Techniques

### The Thoughtful Pause

Before speaking, pause for 2-3 seconds while looking slightly upward. This signals deep thinking and buys time for a buzzword to arrive.

```
Someone: "Should we rewrite the auth service?"

You: [2 second pause, slight upward gaze]
     "That's an interesting question. I think we need to understand
      the full scope of what we're trying to solve before we commit
      to a direction."
```

You have said nothing. You appear to have said something important.

### The Reframe

Never answer the question asked. Answer a better, related question.

```
Question asked:    "Can you finish this by Friday?"
Better question:   "What does 'finish' mean for this deliverable?"
Your answer:       "I want to make sure we're aligned on the
                   definition of done before I give you a timeline."
```

### The Scope Expander

When someone proposes a simple solution, expand the scope until the solution seems premature:

```
Proposal: "Let's just add an index to that table."

You: "I hear you, but I want to make sure we're not solving
     the symptom instead of the root cause. Have we looked at
     the query patterns holistically? This might be part of a
     larger data access story we need to address at the platform level."
```

The index will never be added. The meeting will spawn three more meetings.

### The Action Item Redirect

When an action item is floating toward you, redirect it using one of these:

| Situation | Redirect |
|---|---|
| "Can you do X?" | "I want to make sure the right person owns this — is this more of a [other team] concern?" |
| "You have the context on this." | "I can share context, but I want to make sure we have proper ownership defined." |
| "Can you write this up?" | "Great idea — should this live in the team wiki or the project doc? [Person who suggested wiki] do you want to take a first pass?" |
| Direct assignment | "Happy to be a stakeholder on this. Who's the DRI?" |

## The Follow-Up Meeting Maneuver

This is the most powerful technique. When a decision is about to be made that might create work for you:

```
"I love where this conversation is going. I think we need to
make sure [absent person] is in the room before we finalize
anything — their input would be really valuable here. Can we
schedule a follow-up with the full group?"
```

Requirements for success:
- The absent person must be real
- They must be somewhat relevant
- They must be hard to schedule

This buys 1-2 weeks minimum.

## The Zoom Techniques (Remote-Only)

| Technique | How |
|---|---|
| Strategic mute | Stay muted. Unmute only to agree or ask a clarifying question. |
| The Camera Nod | Nod visibly at whoever is speaking. They feel heard. You are safe. |
| "Sorry, you cut out" | Buys 30 seconds in any situation. Use sparingly (max 2x per meeting). |
| Fake connectivity issue | Nuclear option. Rejoin 90 seconds later. The moment has passed. |
| Background blur | Hides the fact that you are also in another meeting. |
